# Freshet v1 Implementation Plan

> **For the agent team:** This plan is executed by a **4-teammate agent team**
> (`frontend` / `backend` / `integrations` / `review`) — **not** by superpowers
> `executing-plans` or `subagent-driven-development`. Spawn the team with
> `agent-docs/team-prompt-template.md`, handing it this plan + the spec as repo files. The
> lead turns the tasks below into the shared task list. Steps use `- [ ]` so the lead can
> track them. **Phase 0 (contracts) is done first and is the only synchronization barrier;**
> after it lands, the three building verticals run in parallel against the stubs. Each
> teammate works under **plan approval** and expands its section into fine-grained TDD steps.

**Goal:** Ship a functional Freshet v1 — create a stream, research it across free channels via
the detected local agent, synthesize a living markdown document, and on later refreshes surface
only what changed (or stay quiet).

**Architecture:** Tauri 2 desktop app. Rust core (`src-tauri/`) owns the watch engine, file
store, scheduler, and the bridge; an agent-agnostic integrations layer drives the detected local
agent (Claude Code → the `last30days` skill) for research + synthesis; a React/Vite SPA (`src/`)
renders three surfaces over a typed bridge. All work hangs off the shared contracts in spec §5.

**Tech Stack:** Tauri 2, Rust (serde, serde_json, anyhow, tempfile, chrono, tokio), React + Vite
+ TypeScript (vitest, @testing-library/react), the `last30days` Claude skill.

**Spec:** `docs/superpowers/specs/2026-06-12-freshet-v1-design.md` (contracts = §5; acceptance = §2, §9).

coordination (they're the seams).
